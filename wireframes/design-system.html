@Configuration
@EnableWebSecurity
public class SecurityConfig {
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) {
http.csrf().disable()
.authorizeHttpRequests(auth -> auth
.requestMatchers("/api/public/**").permitAll()
.anyRequest().authenticated()
);
return http.build();
}
}

SecurityFilterChain 동작 원리

Spring Security는 서블릿 필터 체인을 기반으로 동작합니다. DelegatingFilterProxy가 서블릿 컨테이너와 스프링 컨텍스트를 연결하는 역할을 합니다.

필터 체인 구조

각 HTTP 요청은 등록된 필터를 순서대로 통과하며, 인증과 인가 과정을 거칩니다.

Spring Security의 핵심은 "누가(Authentication) 무엇을(Authorization) 할 수 있는가"를 결정하는 것입니다.
📘 Note
@EnableWebSecurity 어노테이션을 빠뜨리면 자동 설정이 적용되어 예상과 다른 동작을 할 수 있습니다.
