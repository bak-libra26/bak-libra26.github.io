스프링 시큐리티

JWT 만료·재발급·로그아웃 전략

2026.03.10 · 8분 읽기

🤔 왜 토큰 만료가 필요한가

JWT는 서버에 상태를 저장하지 않는 stateless 인증 방식입니다. 하지만 토큰이 영원히 유효하다면 보안 위험이 커집니다.

🐱
고양이도 이해하는 포인트!
토큰은 열쇠와 같습니다. 열쇠를 분실했을 때를 대비해 유효기간이 있어야 합니다.

Access Token vs Refresh Token

Access Token은 짧은 수명(15분~1시간), Refresh Token은 긴 수명(7일~30일)을 가집니다.

JwtTokenProvider.java 📋 
@Component
public class JwtTokenProvider {
    private final long ACCESS_TOKEN_VALIDITY = 15 * 60 * 1000L;
    private final long REFRESH_TOKEN_VALIDITY = 7 * 24 * 60 * 60 * 1000L;
}

🔄 재발급 전략

클라이언트는 Access Token이 만료되면 Refresh Token으로 새 Access Token을 발급받습니다.

💡 토큰 재발급 시 Refresh Token Rotation을 적용하면 보안을 한 단계 더 강화할 수 있습니다.

🚪 로그아웃 처리

JWT는 stateless이므로 서버에서 직접 토큰을 무효화할 수 없습니다. 블랙리스트 방식을 사용합니다.
