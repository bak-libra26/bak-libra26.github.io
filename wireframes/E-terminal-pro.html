1. 왜 토큰 만료가 필요한가

JWT는 서버에 상태를 저장하지 않는 stateless 인증 방식입니다. 하지만 토큰이 영원히 유효하다면 보안 위험이 커집니다.

Note
토큰은 열쇠와 같습니다. 열쇠를 분실했을 때를 대비해 유효기간이 있어야 합니다.

1-1. Access Token vs Refresh Token

Access Token은 짧은 수명(15분~1시간), Refresh Token은 긴 수명(7일~30일)을 가집니다. 이 두 토큰의 역할 분리가 핵심입니다.

JwtTokenProvider.java copy 
 1@Component
 2public class JwtTokenProvider {
 3    private final long ACCESS_TOKEN_VALIDITY = 15 * 60 * 1000L;
 4    private final long REFRESH_TOKEN_VALIDITY = 7 * 24 * 60 * 60 * 1000L;
 5
 6    public String createAccessToken(Authentication auth) {
 7        return Jwts.builder()
 8            .setSubject(auth.getName())
 9            .setExpiration(new Date(
10                System.currentTimeMillis() + ACCESS_TOKEN_VALIDITY))
11            .signWith(secretKey)
12            .compact();
13    }
14}

2. 재발급 전략

클라이언트는 Access Token이 만료되면 Refresh Token으로 새 Access Token을 발급받습니다. 이 과정에서 보안을 강화할 수 있는 여러 전략이 있습니다.

토큰 재발급 시 Refresh Token Rotation을 적용하면 보안을 한 단계 더 강화할 수 있습니다.

2-1. Refresh Token Rotation

재발급 시 Refresh Token도 함께 새로 발급하는 전략입니다. 기존 Refresh Token은 즉시 무효화합니다.

3. 로그아웃 처리

JWT는 stateless이므로 서버에서 직접 토큰을 무효화할 수 없습니다. 대신 블랙리스트 방식을 사용합니다.
