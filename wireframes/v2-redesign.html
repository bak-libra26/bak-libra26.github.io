Overview

JWT(JSON Web Token)는 당사자 간에 정보를 안전하게 JSON 형태로 전달하기 위한 토큰 기반 인증 방식입니다. 이 글에서는 스프링 시큐리티와 함께 JWT 인증을 구현하는 방법을 단계별로 살펴보겠습니다.

JWT란?

JWT는 Header, Payload, Signature 세 부분으로 구성됩니다. 각 부분은 Base64로 인코딩되어 점(.)으로 연결됩니다.

ⓘ Info
JWT는 RFC 7519에 정의된 개방형 표준(open standard)입니다. 주로 인증(Authentication)과 정보 교환에 사용되며, self-contained 토큰이라는 특징이 있습니다.

구조

JWT 토큰은 다음과 같은 형태를 가집니다:

text copy 
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ1c2VyIn0.signature
|--- Header ---| |--- Payload ---| |-- Sig --|

각 파트의 역할을 정리하면 다음과 같습니다:

구성역할포함 정보
Header토큰 타입과 알고리즘 선언alg, typ
Payload실제 전달할 데이터 (Claims)sub, exp, iat, custom claims
Signature토큰 위변조 검증HMAC or RSA 서명 값

동작 방식

클라이언트가 로그인하면 서버는 JWT를 발급하고, 이후 요청마다 클라이언트는 Authorization 헤더에 토큰을 포함하여 전송합니다.

[ JWT Authentication Flow Diagram ]
960 x 480

JWT 인증의 전체 흐름은 다음과 같습니다:

  1. 클라이언트가 로그인 요청을 보낸다
  2. 서버가 자격 증명을 검증하고 JWT를 발급한다
  3. 클라이언트는 이후 요청에 Authorization: Bearer {token} 헤더를 포함한다
  4. 서버는 JWT를 검증하고 요청을 처리한다
세션 기반 인증과 달리 서버가 상태를 저장하지 않으므로 (stateless), 수평 확장이 용이하고 MSA 환경에 적합합니다.

구현

먼저 JwtTokenProvider 클래스를 생성하여 토큰 생성과 검증 로직을 구현합니다.

java copy 
@Component
public class JwtTokenProvider {

    private final SecretKey secretKey;
    private final long expiration;

    public String generateToken(Authentication auth) {
        return Jwts.builder()
            .setSubject(auth.getName())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(secretKey)
            .compact();
    }
}
▸ Tip
SecretKey는 환경 변수나 외부 설정으로 관리하세요. 코드에 하드코딩하면 보안 취약점이 됩니다.

다음으로 JwtAuthenticationFilter를 구현하여 매 요청마다 토큰을 검증합니다. 이 필터는 OncePerRequestFilter를 상속받아 한 번만 실행되도록 보장합니다.

java copy 
@Override
protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain chain) {

    String token = resolveToken(request);

    if (token != null && tokenProvider.validate(token)) {
        Authentication auth = tokenProvider.getAuthentication(token);
        SecurityContextHolder.getContext().setAuthentication(auth);
    }

    chain.doFilter(request, response);
}
⚠ Warning
JWT는 한번 발급하면 만료 전까지 무효화할 수 없습니다. 토큰 탈취에 대비하여 만료 시간을 짧게 설정하고, Refresh Token 전략을 함께 사용하세요.

마무리

JWT 인증은 stateless한 특성 덕분에 서버 확장이 용이하고, MSA 환경에서 특히 유용합니다. 다음 글에서는 토큰 만료와 재발급 전략에 대해 다루겠습니다.
