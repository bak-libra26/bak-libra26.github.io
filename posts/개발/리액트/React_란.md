# What is React

React is most popluar declarative, component-based state-driven **javascript library for building user interfaces**, created by facebook

- feature of react
	1. Based on components
	2. Declarative
		describe how components look like and how they work using a declarative - telling react what s component should look like, based on current data/state - syntax called **JSX**, a syntax that combines HTML, CSS, Javascript as well as referencing other components, 
	3. State-driven

## Why don't use only `Vanilla Javascript` 

1. Requires lots of direct **DOM manipulation** and **traversing**.
2. Data (stats) is usually stored in the DOM, shared across entrie app.


## WHY DO FRONT-END FRAMEWORKS EXIST ?

1. KEEPING A USER INTERFACE IN SYNC WITH DATA IS REALLY HARD AND A LOT OF WORK.
	-> Front-end frameworks solve this problem and take hard work away from developers.
2. They enforce a corret way of structuring and writing code(therefore contributing to solving the problem of spaghetti code)
3. They give developers and teams a consistent way of buiding front-end applications.


## Summary

1. Rendering components on a webpage (UI) based on ther current state.
2. Keeping the UI in sync with state, by re-rendering (reacting) when state changes.
