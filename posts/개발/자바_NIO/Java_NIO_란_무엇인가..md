# Java NIO

Java에서 입출력(I/O) 처리를 위한 두 가지 핵심 라이브러리가 있다. `java.io` 는 Java 초기부터 제공된 전통적인 입출력 라이브러리로, 파일, 네트워크, 버퍼 등의 입출력을 처리할 수 있게 도와준다. `java.nio` 는 Java 1.4 버전부터 도입된 새로운 입출력 라이브러리로, 기존 java.io의 한계를 극복하고 더 효율적인 입출력 처리를 한다. 

두 라이브러리는 각각 다른 특징을 가지고 있으며, 상황에 따라 어떤 라이브러리를 사용하는 것이 좋은지 달라진다.

`java.nio` 란 1.4 버전부터 도입된 입출력 API로, 기존의 `java.io` 패키지보다 더 효율적이고 유연한 방식으로 파일, 네트워크, 버퍼 등의 입출력을 처리할 수 있게 도와주는 라이브러리이다. `java.io` 라이브러리는 `java.nio` 또한 파일, 네트워크 버퍼 등의 입출력을 처리할 수 있게 도와주는 라이브러리이다.

**기존 IO(java.io)의 특징** 

- **스트림(Stream)기반**: 데이터를 한 방향으로 흐르게 함 (예: InputStream, OutputStream)
- **블로킹 방식**: 입출력 작업이 끝날 때까지 스레드가 멈춰 있음
- **단일 스레드가 하나의 입출력만 처리**: 여러 클라이언트나 파일을 동시에 처리하기 어렵고, 성능에 한계가 있음

**NIO의 특징** 

- **버퍼(Buffer)와 채널Channel)기반**: 데이터를 버퍼에 담고, 채널을 통해 주고받음
- **논블로킹Non-blocking) 지원**: 입출력 작업이 바로 끝나지 않아도 스레드가 멈추지 않음
- **Selector를 통한 멀티플렉싱**: 하나의 스레드가 여러 채널을 동시에 관리할 수 있음

 자바의 Non-Blocking I/O 를 위한 라이브러리이다.
